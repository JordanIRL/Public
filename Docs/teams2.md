# Teams Town halls: controlling what attendees see

Manage the attendee view deliberately. Active-speaker focus isn't enough for a polished all-company event.

## Roles and limits

| Role | Capability | Limit |
|---|---|---|
| Organiser | Schedules, configures, has full production control. Chooses who else gets production tool control | 1 |
| Co-organiser | Same production control as organiser. Standard place to assign the production function | 10 |
| Presenter | Can present and appear on screen. Can be granted production control by the organiser | 100 |
| Attendee | View-only. Mic/camera forced off. Can't rename themselves | 10,000 default |

## Manage what attendees see

Path: **Meeting options > Production tools > Manage what attendees see**. On by default. Confirm the mode before going live.

| Mode | When to use |
|---|---|
| Off | Not for all-company events |
| On *(default)* | Co-organiser brings presenters/content on and off screen. Attendees see changes live |
| On with preview | Queue in Preview feed, then **Send live**. Use for high-stakes events |

Key terms: **Live feed** (what attendees see), **Manage screen pane** (the controls), **Preview feed** (staging area), **Send live** (push to attendees), **Customize screen** (layouts, max 7 video feeds in Dynamic).

## Event setup

| Setting | Recommendation |
|---|---|
| Manage what attendees see | **On** standard, **On with preview** for high-stakes |
| Who can present | Specific named people only |
| Co-organisers | At least one for production, one as backup |
| Production tools control | Grant to the co-organiser running production |
| Green room | On |
| Q&A | On, moderated. Primary attendee interaction channel |
| Town hall chat | Enable if useful; Event group chat exists separately for production crew |
| Recording | On by default; expires after 30 days (extendable to 60) |

## Production workflow

**Before the event**

- Confirm Manage what attendees see mode.
- Confirm the co-organiser has production tools control.
- Pin key presenters in the Manage screen pane.
- Decide the first live view (host only / host + content / content only).
- Choose Customize screen layout.
- Have presenters rehearse window sharing.

**During the event**

- Keep Live feed aligned with the running order.
- Bring presenters/content on and off as needed.
- In On with preview: queue in Preview, then Send live.

**Per transition**

1. Decide next state: speaker / content / speaker + content / panel.
2. Queue in Preview (if using preview mode).
3. Send live.
4. Confirm Live feed matches intent.
5. Remove anything no longer needed.
